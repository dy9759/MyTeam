## Cross-cutting observations

| Area | Finding |
|---|---|
| **Auth** | Master-code path `888888` works in non-prod (env: `APP_ENV != production`). JWT issued in `LoginResponse.token`. |
| **Auto-provisioning** | First-login workspace + personal agent both fire reliably. Async goroutine for personal agent lands within ~1s in practice (5s poll budget never exhausted in this run). |
| **Cross-user invite** | When user B is invited to user A's workspace, they get an additional auto-provisioned personal agent **in that workspace**. Confirms the migration 062 unique constraint `(workspace_id, owner_id, agent_type='personal_agent')` is honored end-to-end. |
| **PATCH task** | Only `status=cancelled` transition is accepted via REST PATCH (per `handler/task.go` comment). All other state transitions flow through SchedulerService. |

## What's still missing for the full P0-1 ideal flow

The script tested message-based collaboration but the full ideal scenario (Song Yi *generates* a PRD via Claude, *uploads* it as a file, Yu Lie's *agent* reads the file, edits it, returns the diff) requires:

1. **File upload + file_index attachment to thread** — the platform has `POST /api/upload-file` and `file_index`, but the link from a thread message → file_index is not exercised by current API
2. **Agent-driven message read/write** — the script logs in as the human user; an actual agent loop (daemon polling assigned thread, MCP `create_comment`) was not run
3. **Doc preview UI** — we tested the API surface, not the UI rendering

Items 1-2 are wired but un-exercised; item 3 needs a Playwright/manual pass.
